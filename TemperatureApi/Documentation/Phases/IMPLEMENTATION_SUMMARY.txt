RATE LIMITING
Before: config.PermitLimit = 20 (blocks 99%+ of requests)
After:  config.PermitLimit = 100,000 (supports high scale)
Code:
  options.AddFixedWindowLimiter("fixed", config =>
  {
      config.PermitLimit = 100000;
      config.Window = TimeSpan.FromSeconds(10);
      config.QueueLimit = 50000;
  });

REQUEST TIMEOUTS
Added: CancellationTokenSource with 30-second timeout
Code:
  app.Use(async (context, next) =>
  {
      using var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));
      // ... handle timeout ...
  });

ASYNC LOGGING
Before: .WriteTo.File(...) [blocking I/O]
After:  .WriteTo.Async(a => a.File(...)) [non-blocking]
Code:
  .WriteTo.Async(a => 
      a.File("logs/temperature-api-.log", 
          rollingInterval: RollingInterval.Day,
          retainedFileCountLimit: 30))

💡 KEY INSIGHTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Rate Limiting was BLOCKING 99.99% of traffic
   Solution: Increased from 20 to 100,000 per 10 seconds
   Lesson: Rate limiting should be at gateway level, not API level

2. Database Connection Pool was UNDERSIZED
   Solution: Configured 10-500 connection pool
   Lesson: Default PostgreSQL limits (30 connections) insufficient

3. Blocking Logging was KILLING THROUGHPUT
   Solution: Switched to async file writing
   Lesson: File I/O blocks request threads, async is critical

4. No Request Timeouts ALLOWED HANGING
   Solution: Added 30-second timeout
   Lesson: Timeouts prevent cascading failures

5. No Resource Limits COULD CRASH HOST
   Solution: Set CPU and memory limits
   Lesson: Containers must have resource constraints

6. Hardcoded Secrets were SECURITY RISK
   Solution: Moved to environment variables
   Lesson: Configuration should be external

7. PostgreSQL NOT TUNED for scale
   Solution: Added connection limits, memory config
   Lesson: Database defaults not production-ready

8. No Configuration Validation
   Solution: Added JWT key validation
   Lesson: Fail fast with clear errors
