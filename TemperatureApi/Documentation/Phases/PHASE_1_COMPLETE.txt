1. Rate Limiting
   ✓ Wrong limits can block 99%+ of legitimate traffic
   ✓ Best placed at API Gateway/Load Balancer, not at API
   ✓ Current fix: Moved to 100,000 per 10s for proper throughput

2. Database Connections
   ✓ Default 30 connections exhausts quickly at scale
   ✓ 500 connection pool supports thousands of users
   ✓ Scaling: Load balance across multiple API instances

3. Request Timeouts
   ✓ Missing timeouts cause resource exhaustion
   ✓ Slow queries can hang indefinitely
   ✓ 30-second timeout is good default for most APIs

4. Async Logging
   ✓ File I/O blocks request threads, kills throughput
   ✓ Async sinks prevent this bottleneck
   ✓ 10-20% performance improvement possible

5. Resource Limits
   ✓ Containers without limits can crash host system
   ✓ CPU limits prevent runaway processes
   ✓ Memory limits prevent OOM kills

6. Configuration Management
   ✓ Hardcoded secrets are security risk
   ✓ Environment variables enable proper DevOps
   ✓ Configuration validation prevents runtime surprises

7. PostgreSQL Tuning
   ✓ Default config not suitable for high throughput
   ✓ Connection limits, memory tuning essential
   ✓ Monitoring queries for optimization

8. Testing at Scale
   ✓ Single machine tests don't catch all issues
   ✓ Load testing reveals real bottlenecks
   ✓ Distributed testing closer to production reality
