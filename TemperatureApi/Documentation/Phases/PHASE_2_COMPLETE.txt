AVAILABLE METRICS:

1. REQUEST RATE
   Metric: http_requests_received_total
   Unit: Counter (total requests since start)
   Usage: Calculate requests/second over time
   Example: rate(http_requests_received_total[5m])

2. REQUEST DURATION
   Metric: http_request_duration_seconds
   Unit: Histogram (buckets: 0.005, 0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1, 2.5, 5, 10)
   Usage: Calculate latency percentiles
   Example: histogram_quantile(0.99, http_request_duration_seconds_bucket)

3. REQUEST SIZE
   Metric: http_request_size_bytes
   Unit: Histogram (request body size)
   Usage: Monitor bandwidth usage
   Example: rate(http_request_size_bytes_sum[5m]) / rate(http_request_size_bytes_count[5m])

4. RESPONSE SIZE
   Metric: http_response_size_bytes
   Unit: Histogram (response body size)
   Usage: Monitor bandwidth usage
   Example: rate(http_response_size_bytes_sum[5m]) / rate(http_response_size_bytes_count[5m])

5. STATUS CODE DISTRIBUTION
   Metric: http_requests_received_total (with "status" label)
   Unit: Counter
   Usage: Track 2xx, 4xx, 5xx errors
   Example: http_requests_received_total{status="500"}

ALL METRICS INCLUDE LABELS:
  • method: GET, POST, PUT, DELETE, etc.
  • path: /api/temps, /api/auth/login, etc.
  • status: 200, 400, 401, 404, 500, etc.
